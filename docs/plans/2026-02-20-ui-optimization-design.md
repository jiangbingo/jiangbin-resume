# 个人简历网站 UI 优化设计方案

> 创建日期: 2026-02-20
> 方案类型: 渐进增强（基于现有 Astro 架构）

## 概述

本方案旨在通过简约专业的设计风格，对个人简历网站进行全面的 UI 和用户体验优化，包括：

- Hero 区域视觉升级与协调微动画
- 导航增强（滚动高亮、平滑锚点、移动端菜单）
- 卡片精致交互动效
- 功能增强（技能等级可视化、项目详情弹窗）

## 设计原则

- **简约专业**: 干净留白、微动画、精致排版、强调内容本身
- **协调统一**: 所有动画效果保持克制和风格一致
- **渐进增强**: 优先使用 CSS 动画，保持轻量和快速加载
- **移动优先**: 确保移动端体验与桌面端同样优秀

## 一、Hero 区域视觉升级

### 1.1 布局结构

```
┌─────────────────────────────────────────────────┐
│                 [粒子背景 - 优化]                  │
│                                                 │
│              ┌─────────────┐                    │
│              │   头像      │  ← 光晕呼吸动画      │
│              │  (圆形)     │    悬停微缩放       │
│              └─────────────┘                    │
│                                                 │
│           江 斌  ← 逐字渐入动画                   │
│                                                 │
│    AI 应用工程师 / Python 后端开发工程师          │
│         ← 打字机效果循环展示不同职位              │
│                                                 │
│           📍 杭州 / 武汉                         │
│                                                 │
│      [GitHub]  [微信]  [Email]                  │
│         ← 按钮悬停上浮 + 图标动画                │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 1.2 动效规格

| 元素 | 动画效果 | 实现方式 |
|------|----------|----------|
| **头像** | 柔和光晕呼吸（3s 周期）+ 悬停缩放 1.05 | CSS keyframes + transition |
| **姓名** | 页面加载时逐字渐入（0.1s 间隔） | CSS animation + stagger |
| **职位** | 打字机效果，循环展示 3 个职位 | CSS + 少量 JS |
| **按钮组** | 悬停上移 4px + 阴影加深 | CSS transition |
| **粒子** | 保持现有，调整密度和速度 | 现有 Particles 组件 |

### 1.3 配色

- 主色：`slate-900` / `slate-100`（深浅模式）
- 点缀：`blue-600` → 添加 `indigo-500` 作为渐变点缀

## 二、导航增强

### 2.1 桌面端导航

```

### 2.2 移动端导航

```
┌───────────────────────┐          ┌───────────────────────┐
│  江斌          [☰]    │    →     │  ✕                   │

### 2.3 平滑滚动锚点

| 导航项 | 锚点 | 滚动行为 |
|--------|------|----------|
| 首页 | `#hero` | 滚动到顶部 |
| 核心能力 | `#skills` | 平滑滚动 + 导航高亮 |

| 功能 | 技术 |
|------|------|
| 滚动高亮 | Intersection Observer API |
| 平滑滚动 | CSS `scroll-behavior: smooth` |
| 移动端菜单 | CSS transform + JS toggle |
| 导航栏透明度 | scroll event + CSS transition |

## 三、卡片交互动效

### 3.1 技能卡片

```
正常状态                          悬停状态
┌─────────────────┐              ┌─────────────────┐
│                 │              │  ↑ 上浮 8px     │
│   🤖            │      →       │  🤖             │
│                 │              │                 │
│  AI 应用工程    │              │  AI 应用工程    │
│                 │              │  ─────────────  │ ← 分隔线渐入
│  [RAG] [Prompt] │              │  [RAG] [Prompt] │
│  [Agent] [...]  │              │  [Agent] [...]  │ ← 标签微放大
│                 │              │                 │
│                 │              │  [→ 查看详情]   │ ← 新增按钮渐入

**动画时序**：
- 上浮 + 阴影：`0.3s ease-out`

- 内容渐入：`0.2s delay` 后触发

│                                     │

│  └─────────────┘ └─────────────┘   │
│                                     │
│          [ 查看详情 → ]             │  ← 悬停时渐入
└─────────────────────────────────────┘
```

### 3.3 滚动进入动画

卡片进入视口时触发渐入动画：

```
卡片初始状态              滚动进入后
    ↓                        ↓
opacity: 0             opacity: 1
translateY: 30px  ──→  translateY: 0
scale: 0.95            scale: 1

触发条件：卡片顶部进入视口 80% 位置
动画时长：0.5s ease-out
交错延迟：每张卡片延迟 0.1s
```

### 3.4 CSS 实现要点

```css
/* 卡片基础过渡 */
.skill-card {
  transition: transform 0.3s ease-out,
              box-shadow 0.3s ease-out,
              border-color 0.4s ease;
}

.skill-card:hover {
  transform: translateY(-8px);
  box-shadow: 0 20px 40px rgba(0,0,0,0.1);
}

/* 滚动进入动画 */
.card-enter {
  opacity: 0;
  transform: translateY(30px);
}
.card-enter.visible {
  opacity: 1;
  transform: translateY(0);
  transition: all 0.5s ease-out;
}
```

## 八、性能考量

| 方面 | 策略 |
|------|------|
| **动画** | 优先使用 CSS 动画，避免 JS 动画库 |
| **滚动检测** | Intersection Observer 优于 scroll event |
| **图片** | 项目图片使用 Astro Image 优化 |
| **JS 体积** | 零运行时框架，仅交互组件加载 JS |
