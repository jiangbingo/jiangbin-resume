## 什么是真正的"会用 AI"？

很多人以为会用 AI，就是会写提示词，会在对话框里聊天。实际上提示词不用你来写，AI 写得更好更专业。对话框聊天始终太碎片化，无法形成系统。

**真正的"会用"，是让 AI 替你干活。**

不要把 AI 当成聊天对象，要把它当成你头脑风暴的拍档，当成你的 CTO。你要做的，是让 AI 写出工具，把你每天重复枯燥的工作接管过去，让 AI 把你的想法产品化。

## 接触 Vibe Coding 必会上瘾

我本科学的通信工程，主要研究的是给芯片写代码（嵌入式开发）。我永远记得最开始用代码点亮单片机上的小灯泡的那一刻带来的快感。

**所想即所得**，你敲下一行代码，现实世界立刻给你一个亮灯的反馈。从此，通宵写代码成为了常态。

### 学习最快的方式：极速反馈

以前学东西是"背书 → 考试 → 等分数"，周期太长。

Vibe Coding 是"想法 → AI 生成 → 看到结果"，就像玩游戏一样，几秒钟一个回合。这种快感会推着你不断往前走。

> 干中学，即时反馈，是学习最快的方式。

## 你的新身份：不是程序员，是产品经理

别去想怎么写代码，用什么编程语言来写，那不是你的事。

在 Vibe Coding 的世界里，分工极其明确：

| 角色 | 职责 |
|------|------|
| AI | 负责"怎么做"（写代码、改 BUG） |
| 你 | 负责"做什么"（提需求、挑毛病） |

你不需要懂代码的语法，但你需要懂你自己的需求。

> 你能把需求描述得多清楚，AI 就能做得多好。

## 最大的坑：AI 的"金鱼记忆"

为什么复杂的项目容易做烂？

因为大模型有上下文长度的限制，AI 记性不好。你跟它聊得太久，它就忘了最开始你要干什么，然后开始胡编乱造（我们叫"幻觉"）。

### 解决办法

不要只靠嘴说，要靠文档记。

Manus 和 Claude Code 在上下文管理方面做得很好，因为它们懂得看"文件"。文件就是 AI 的外挂硬盘，帮它记住所做的一切。

**上下文只负责处理当前任务，处理完后丢到文件中保存起来，清空记忆，开始新的工作。需要唤醒记忆时，再去文件中读取。**

> Vibe Coding 不是对话框交互式开发，而是工程级别的上下文管理。

## 核心方法：搭积木逻辑

新手怎么做复杂的东西？千万不要试图一口吃成胖子。

### 搭积木思维

1. **拆解**：别让 AI 做"微信"，让它先做"一个输入框"
2. **只看结果**：代码写得怎么样你别管，你只需要试一下能不能用
   - 能用 → 这个积木是好的 → 保留
   - 不能用 → 扔回去让 AI 重做 → 修正
3. **拼装**：把一个个验证好的"积木"拼起来，就是你的产品

> 先局部，后整体；先跑通，后重构。这就是 Vibe Coding 的核心心法。

## 工具与战术

不用纠结选哪个工具最好，根据你的预算和投入度，选择最适合的"战术组合"。

### 方案 A：零成本体验版

适合验证想法，用免费工具跑通最小流程。

| 工具 | 角色 | 说明 |
|------|------|------|
| ChatGPT | 大脑/产品经理 | 头脑风暴，拆解需求 |
| Google Antigravity | 车间/开发环境 | 每天有免费额度 |
| Gemini CLI | 替补员工 | 免费额度接力 |

### 方案 B：高性价比实战版

每月 $20 的投入，换取工业级的精准度。

| 工具 | 角色 | 说明 |
|------|------|------|
| ChatGPT Plus | 主力 | $20/月 |
| CodeX CLI | 特种兵 | 包含在会员中 |
| Cursor/VS Code | IDE | 免费版足够 |

**双流操作技巧**：
- 终端 A (Gemini 免费版)：处理脏活累活
- 终端 B (CodeX 付费版)：处理精细活

### 方案 C：终极极客版

组建一支特种部队，让每个 AI 发挥其长板。

| AI | 角色 | 职责 |
|-----|------|------|
| ChatGPT | 战略顾问 | 头脑风暴，输出需求文档 |
| Gemini | 审核员 | 评审需求，查漏补缺 |
| Claude Code | 架构师 | 搭建框架，管理上下文 |
| CodeX | 特种兵 | 精准执行代码编写 |

## 最后的建议

不要问我"怎么安装这些工具"。你已经进入 AI 时代了。

打开 ChatGPT 或 Gemini，问它：

> "我是 Mac/Windows 系统，请一步步教我怎么安装 CodeX CLI。"

如果连这一步都需要人类手把手教，那你可能还没准备好开始 Vibe Coding。
